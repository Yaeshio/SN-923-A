# UI/UX仕様書 (Draft)

## 1. アプリケーションフローとエントリーポイント
本アプリケーションのエントリーポイント（ホーム画面・初期表示）は **プロジェクト管理画面 (`ProjectMasterUI`)** とします。
製造の進行や管理はプロジェクトおよびユニットに強く紐づくため、ユーザーはまず全体コンテキストとなるプロジェクトを選択し、その詳細画面から「ユニット管理」「設計データ登録（インポート）」「工程管理ボード」などに遷移する業務構造を基本とします。

## 2. 画面一覧

| 画面名 | ID | 概要 |
| :--- | :--- | :--- |
| インポート画面 | `ImportUI` | STLファイルを用いた設計データの一括登録。3Dプレビューとドラッグ＆ドロップ対応。 |
| 製造指示画面 | `OrderUI` | サジェスト・フィルタ付きコンポーネントを用いた部品の選択と製造個体（PartItem）の生成。 |
| 工程管理ボード | `BoardUI` | 部品個体のステータスを一覧・操作するメインUI。高度なフィルタリングと一括操作をサポート。 |
| 部品詳細画面 | `DetailUI` | 個別の部品情報の確認、STLファイルのダウンロード。 |
| プロジェクト管理画面 | `ProjectMasterUI` | プロジェクトのマスタ管理（CRUD操作）。 |
| ユニット管理画面 | `UnitMasterUI` | ユニットのマスタ管理（プロジェクトと連携したCRUD操作）。 |

## 3. 画面詳細

### 2.1 インポート画面 (`ImportUI`)
- **構成**: マスタデータ検索ドロップダウン、ドラッグ＆ドロップエリア、ファイル名解析と3Dプレビュー。
- **特徴**:
    - 所属する **プロジェクトおよびユニット** を検索・選択できるUI（Combobox）の導入。
    - 画面全体のドラッグ＆ドロップによるインポート直感操作。
    - Three.js等を用いたSTLモデルの視覚的プレビュー機能で登録前確認。
    - ファイル名からの部品番号自動抽出および自動データ生成。

### 2.2 製造指示画面 (`OrderUI`)
- **構成**: リッチなリスト情報を持つ選択UIと製造個体作成フォーム。
- **特徴**:
    - 大規模データに対応するため、検索機能付きの選択コンポーネント（React Select等）を利用。
    - 部品のサムネイル画像や製造機（Machine）のステータス（カラーバッジ付きで稼働状況を表示）を選択候補に組み込み、誤操作を防止。

### 2.3 工程管理ボード (`BoardUI`)
- **構成**: 高度な絞り込み機能を備えた、かんばん（Swimlane）とテーブル（リスト）のハイブリッドUI。
- **ステータス列**: `READY`, `PRINTING`, `CUTTING`, `SANDING`, `INSPECTION`, `COMPLETED`, `SHIPPED`, `DISCARD`
- **特徴**:
    - ドラッグ＆ドロップによる直感的なステータス遷移（かんばん方式）。
    - 複数個体を同時に操作可能なバルク処理（一括ステータス変更用テーブルモード）。
    - 「破棄」や「出荷」などの重大な状態変更時における確認モーダルと理由入力ダイアログ。
    - ステータス、部品名、製造機等による多段フィルタリングおよびソート機能。

### 2.4 プロジェクト・ユニット管理画面 (マスタ設定・新規追加)
- **構成**: 一覧画面およびCRUD操作用フォーム（モーダルまたは専用ページ）。
- **特徴**:
    - データベースを直接操作せずに運用できるマスタ管理機能を提供。
    - プロジェクト詳細画面からユニットを直接追加・編集できる、階層構造を意識したナビゲーション。

## 4. 全体的なUX方針
- **レスポンシブと現場端末への最適化**: 工場内のタブレット端末や共有タッチパネルを想定し、タップ領域（ボタン・選択肢）を大きく確保したタッチファーストな設計。
- **リアルタイム同期**: 複数人利用環境下における画面上の遅延・競合を防ぐため、他操作の即時反映（Supabase Realtime等による）を導入。
- **確実なフィードバック**: 通信状態やエラーのトースト通知、非同期処理時のスケルトンスクリーンやローディング表示の徹底により、利用者の不安を軽減。
